# 주요 사용 흐름

## 1. 비로그인 사용자 콘텐츠 탐색

1. 사용자가 홈 화면에 진입한다.
2. 지역 또는 카테고리를 선택한다.
3. 콘텐츠 목록을 탐색한다.
4. 상세 정보를 확인한다.
5. 마음에 드는 콘텐츠를 선택 목록에 담는다.

비로그인 사용자는 콘텐츠 탐색과 일정 생성 전 단계까지 자연스럽게 경험할 수 있어야 한다.

## 2. 일정 생성 흐름

1. 사용자가 선택 목록을 확인한다.
2. 여행 날짜, 기간, 동행 유형 등 조건을 입력한다.
3. 클라이언트가 필수값과 선택 콘텐츠 수를 검증한다.
4. 서버의 일정 생성 API를 호출한다.
5. 로딩 상태를 표시한다.
6. 성공 시 생성된 일정을 타임라인으로 표시한다.
7. 실패 시 사용자에게 재시도 가능한 원인과 다음 행동을 안내한다.

## 3. 로그인 사용자 저장 흐름

1. 사용자가 생성된 일정에서 저장을 선택한다.
2. 인증 상태를 확인한다.
3. 로그인되어 있으면 저장 API를 호출한다.
4. 로그인되어 있지 않으면 로그인 화면 또는 OAuth 시작 흐름으로 이동한다.
5. 저장 성공 후 내 일정 목록에서 다시 확인할 수 있게 한다.

## 4. 공유 일정 조회 흐름

1. 사용자가 공유 링크로 진입한다.
2. 공유 토큰 또는 공유 ID로 공개 일정 API를 호출한다.
3. 일정 소유자, 지역, 날짜, 장소 목록을 읽기 전용으로 표시한다.
4. 만료되었거나 존재하지 않는 공유 링크는 명확한 오류 상태를 표시한다.

## 5. 오류와 빈 상태

- 콘텐츠가 없으면 필터를 바꾸거나 다른 지역을 선택하도록 안내한다.
- 서버 연결 실패 시 재시도 버튼을 제공한다.
- 인증 만료 시 로그인 재요청 흐름으로 전환한다.
- 일정 생성 실패 시 선택 콘텐츠와 여행 조건을 유지해 사용자가 다시 입력하지 않게 한다.
