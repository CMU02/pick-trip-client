# 핵심 기능

Pick Trip Client는 Pick Trip 서비스의 웹 사용자 경험을 담당한다. 기능은 지역 콘텐츠 탐색, 선택한 콘텐츠 기반 일정 생성, 생성 결과 확인과 공유 흐름을 중심으로 설계한다.

## 1. 지역 콘텐츠 탐색

- 하동, 영주, 예천의 음식, 축제, 관광지, 문화, 자연 콘텐츠를 카드 또는 목록 형태로 제공한다.
- 지역, 카테고리, 키워드, 운영 상태 같은 필터를 통해 원하는 콘텐츠를 빠르게 찾을 수 있게 한다.
- 각 콘텐츠는 이름, 지역, 카테고리, 대표 이미지, 요약 설명, 위치 정보, 운영 정보 등 사용자가 일정에 넣을지 판단할 수 있는 정보를 우선 노출한다.

## 2. 콘텐츠 선택

- 사용자가 마음에 드는 콘텐츠를 선택 목록에 담을 수 있게 한다.
- 선택 목록은 일정 생성의 입력값이므로 중복 선택, 삭제, 우선순위 조정이 가능해야 한다.
- 선택 수가 부족하거나 지역이 섞여 일정 생성이 어려운 경우 명확한 안내를 제공한다.

## 3. 여행 조건 입력

- 여행 날짜, 기간, 동행 유형, 이동 방식, 선호 강도 같은 조건을 입력받는다.
- 서버 API 계약에 맞는 request DTO 형태로 값을 정규화한다.
- 필수값 누락과 잘못된 날짜 범위는 클라이언트에서 먼저 검증한다.

## 4. AI 일정 생성 결과 표시

- 서버가 생성한 일정 결과를 날짜별, 시간대별, 장소별 흐름으로 표시한다.
- 장소 간 이동, 식사 시간, 대체 후보, 유의사항이 있다면 사용자가 한눈에 확인할 수 있게 구성한다.
- 생성 실패나 일부 콘텐츠 제외 사유는 숨기지 않고 사용자에게 설명한다.

## 5. 일정 저장과 공유

- 로그인 사용자는 생성된 일정을 저장하고 다시 조회할 수 있다.
- 공유 가능한 일정은 읽기 전용 화면으로 제공한다.
- 인증이 필요한 액션은 로그인 유도 흐름을 제공하되, 콘텐츠 탐색 자체는 막지 않는다.

## 6. 반응형 웹 경험

- 모바일, 태블릿, 데스크톱에서 콘텐츠 탐색과 일정 확인이 모두 가능해야 한다.
- 여행 중 현장에서 볼 수 있는 화면은 모바일 우선으로 설계한다.
- 지도, 카드, 필터, 일정 타임라인은 좁은 화면에서도 조작 가능한 크기와 간격을 유지한다.
