# MVP 범위

## 목표

MVP는 사용자가 지역 콘텐츠를 탐색하고, 원하는 콘텐츠를 선택한 뒤, 서버의 AI 일정 생성 API를 통해 여행 일정을 확인하는 흐름을 완성하는 것이다.

## 포함 범위

### 콘텐츠 탐색

- 하동, 영주, 예천 지역 선택
- 음식, 축제, 관광지, 문화, 자연 카테고리 표시
- 콘텐츠 목록과 상세 정보 표시
- 기본 필터와 검색

### 콘텐츠 선택

- 선택 목록 추가/삭제
- 선택 개수 표시
- 일정 생성 전 선택 목록 검증

### 일정 생성

- 여행 날짜와 기간 입력
- 동행 유형 또는 선호 조건 입력
- 일정 생성 API 호출
- 생성 결과 타임라인 표시
- 생성 실패와 로딩 상태 처리

### 인증 연동

- 로그인 필요 액션에서 인증 유도
- 인증 토큰이 필요한 API 호출 경계 정의
- 인증 만료 시 재로그인 안내

### 저장과 공유

- 로그인 사용자 일정 저장
- 저장된 일정 조회
- 공유 링크 일정 조회

## 제외 범위

- 자체 회원가입과 비밀번호 기반 로그인
- 실시간 지도 경로 최적화
- 결제, 예약, 쿠폰
- 다국어 지원
- 관리자 콘텐츠 편집 UI
- 오프라인 모드
- 푸시 알림

## 품질 기준

- `bun run lint`와 `bun run build`가 통과해야 한다.
- 모바일 화면에서 핵심 플로우가 끊기지 않아야 한다.
- API 실패, 빈 목록, 로딩 상태가 모두 화면에 표현되어야 한다.
- 서버 secret이 클라이언트 번들에 포함되지 않아야 한다.
