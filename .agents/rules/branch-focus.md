# 브랜치 포커스

## 목적

한 브랜치에서 너무 많은 변경이 섞이면 리뷰와 롤백이 어려워진다. 브랜치는 하나의 목표만 가진다.

## 기본 규칙

- 브랜치 이름과 관련 없는 변경을 포함하지 않는다.
- 문서, 설정, 기능, 리팩터링을 불필요하게 한 PR에 섞지 않는다.
- 작업 중 발견한 별도 문제는 새 이슈나 새 브랜치 후보로 기록한다.
- 사용자 요청 범위를 벗어난 리팩터링은 진행하지 않는다.

## 허용되는 변경 예시

콘텐츠 카드 목록 이슈의 `feat/12` 브랜치:

- 콘텐츠 카드 컴포넌트 추가
- 콘텐츠 목록 API 타입 추가
- 콘텐츠 목록 화면 연결
- 해당 기능을 위한 최소 스타일 추가

## 피해야 할 변경 예시

같은 `feat/12` 브랜치에서 다음 변경은 분리한다.

- 전체 폴더 구조 재편
- Biome 설정 변경
- 인증 플로우 수정
- README 대규모 수정
- unrelated dependency 추가

## 예외

다음은 현재 작업을 완료하기 위해 필요한 경우 함께 포함할 수 있다.

- 새 기능을 빌드하기 위한 최소 타입 추가
- 기능 동작에 필요한 작은 공통 유틸 추가
- 관련 문서의 짧은 업데이트
- 변경 파일에 직접 영향을 주는 린트 수정

## 중단 조건

작업 중 다음 상황이 생기면 범위를 재확인한다.

- 수정 파일이 예상보다 크게 늘어난다.
- API 계약 변경이 필요하다.
- 인증 또는 보안 정책 변경이 필요하다.
- 패키지 추가가 필요하다.
- 기존 화면의 동작이 바뀐다.
